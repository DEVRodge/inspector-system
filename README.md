## Inspector System

一个前端为主的「巡检 / 检查」系统项目仓库，目前主要用于前端功能开发与验证，后端接口将由其他服务提供。

### 功能概览（当前规划）

- **前端功能开发**：基于模拟数据完成页面和交互逻辑，实现完整的功能闭环。
- **接口对接占位**：保留统一的接口封装层，后续可无缝切换为真实后端接口。
- **组件复用**：抽象通用组件，方便在不同检查场景下复用。

### 本地开发

1. **克隆仓库**

   ```bash
   git clone https://github.com/DEVRodge/inspector-system.git
   cd inspector-system
   ```

2. **安装依赖**

   假设项目使用 Node.js + 前端框架（如 React / Vue 等），在仓库根目录执行：

   ```bash
   pnpm install
   # 或
   npm install
   # 或
   yarn
   ```

3. **启动开发环境**

   ```bash
   pnpm dev
   # 或
   npm run dev
   # 或
   yarn dev
   ```

4. 在浏览器中访问文档或控制台输出提示的地址（一般为 `http://localhost:5173` 或 `http://localhost:3000`）。

### 接口与数据约定

- **当前阶段**：使用前端模拟数据（mock）完成页面联动与流程验证。
- **未来阶段**：当你提供后端接口文档或要求对接时，将：
  - 在保持现有前端交互不变的前提下，对接真实接口；
  - 支持环境切换（本地 / 测试 / 生产）；
  - 保留必要的 mock 能力，便于开发和联调。

### 开发约定

- **只写前端逻辑**：接口实现由后端同学负责，当有接口文档时，再进行集成和联调。
- **优先保证功能闭环**：通过模拟数据、前端路由和状态管理，先把业务流程跑通。
- **代码风格**：建议统一使用 ESLint + Prettier（若你有既定规范，可以后续补充到本 README）。
